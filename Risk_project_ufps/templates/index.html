¿Qué es RISKO?

Sofware para la gestión de riesgos en proyectos basado en las mejores practicas de estandares internacionales para la dirección de proyectos, herrmientas para la gestión de riesgos disponibles en el mercado y aportes de diferentes directores de proyectos.

Acerca de RISKO

RISKO nace como un producto de formación del trabajo de investigación titulado "SOFTWARE PARA LA GESTIÓN DE RIESGOS BASADO EN LAS MEJORES PRÁCTICAS DE LA ADMINISTRACIÓN DE PROYECTOS" del grupo de investigación GIDIS de la Universidad Francisco de Paula Santander en Cúcuta, Colombia.

Servicios

Servicios

RISKO ofrece una cantidad de servicios acorde a los procesos abordados para la gestión de riesgos en proyectos, donde se incluye la planificación, identificación, evaluación, planificar respuestas, controlar y comunicar.

Planificar

Mediante este proceso se busca realizar un plan estrategico para gestionar los riesgos de su proyecto, donde podra visualizar la EDR de su proyecto, administrar su equipo de trabajo, y definir las clasicaciones de probabilidad, impacto y evaluación del riesgo.

Identificar

Este proceso busca identificar los riesgos individuales y generales del proyecto, para esto, se podran registrar nuevos riesgos o los sugeridos por el sistema, asi mismo, se les podra asignar un reponsable, acciones de respuesta preliminares, y actividades del cronograma del proyecto.

Evaluar

Mediante este proceso se busca poder agregar una evaluación a los riesgos del proyecto, para esto podra asignar un valor de impacto y probabilidad deseado y el sistema calculara los resultados, los cuales podran ser observados mediante una matriz de impacto y probabilidad.

Planificar Respuestas

Este proceso busca crear un plan de respuesta a los riesgos, donde se podran asociar acciones de respuesta, tareas, y recursos para tratar estos. Adicionalmente al terminar de realizar este plan, se podra generar una linea base de la gestión de riesgos para poder empezar a controlar.

Controlar

Este proceso tiene por objetivo el monitoreo constante del plan de respuestas desarrollado y la administración de las fechas, estados y observaciones de las tareas segun la ejecución que vaya teniendo el proyecto, para estp el software presenta esa información mediante un cronograma de facil uso.

Comunicar

Este proceso tiene por objetivo poder comunicar el estado actual de la gestión de riesgos del proyecto al equipo del proyecto, para esto el software permite generar reportes actualizados de todos los procesos de la gestión de riesgos en formato Excel.

Cierre del Proyecto

Mediante este proceso se podra notificar la finalización de la gestión de riesgos del proyecto, asi como administrar las lecciones aprendidas que fueron surgiendo durante la ejecución del mismo.
